როგორ დავიწყო bug bounty hunting-ი? როგორ გავაუმჯობესოთ უნარები?

ეს არის ის მარტივი ნაბიჯები, რომლებიც ყველა დამწყებმა Bug Hunter-მა უნდა გამოიყენოს უნარების გასაუმჯობესებლად:

ისწავლე რათა გააკეთო ეს; შემდეგ შეუდექი გატეხვას!

Hacker-ების ძირითადი აზრი და სურვილი წარმოადგენს გაიგოს მეტი რათა დააექსპლოიტონ და აღმოაჩინონ ახალი სისუსტეები, Hacker-ები სწავლობენ იმის შექნმას რაც გამიზნული აქვთ. ამის კეთებით Hacker-ები უკეთესად ერკვევიან თანდათანობით თუ სად ჩნდება პრობლემური სისუსტეები. მაგალითად, როდესაც ხალხი მეკითხება თუ როგორ შევასრულოთ Subdomain Takeover, პირველ რიგში მე დავრწმუნდე უნდა რომ მათ იციან თუ რა არის Domain Name System (DNS) და ვთავაზობ შექმნან საკუთარი დომენი რათა შეასრულონ მსგავსი სახის შეტევა.

წიგნების კითხვა. ბევრი წიგნების კითხვა.

ერთერთი გზაა გახდე უკეთესი წაიკითხო რეგულარულად Hunter-ების ნაშრომები და 'hackers'-ების write-ups-ები. მიყვეთ /r/netsec და Twitter ფანტასტიური write-ups-ებს სადაც იდება სხვა და სხვა სახის უსაფრთხოების თემები, რაც დაგეხმარებათ მოტივაციაში და გაგიუმჯობესებს უნარებს.წასაკითხად კარგი წიგნების ჩამონათვალი, გთხოვთ მიმართოთ "რომელი წიგნებია აუცილებლად წასაკითხი?".

შემოგვიერთდით დისკუსიაში და დასვით კითხვები.

როგორც იცით ინფორმაციული უსაფრთხოების საოგადოება მდიდარია საინტერესო დისკუსიებით დაწყებული დარღვევებით დამთავრებული თვაყურის დევამდე ასევე სხვა.Bug bounty საზოგადოება შეადგენს hunter-ებს, უსაფრთხოების ანალიტიკოსებს და პლატფორმის პერსონალს რომელიც ეხმარება ამა თუ იმ მხარეს გააუმჯობესონ ის რითაც არიან დაკავებული. არსებობს 2 პოპულარული და ცნობილი bug bounty ფორუმები: Bug Bounty Forum და Bug Bounty World.

მიიღეთ მონაწილეობა და შეიტანეთ წვლილი open source პროექტებში; ისწავლეთ კოდინგი.

გადადით https://github.com/explore ან https://gitlab.com/explore/projects და აირჩიეთ პროექტი რომელსაც მხარს დაუჭერთ და შეიტანთ საკუთარ წვლილს მის განვითარებაში. მსგავსი ქცევით თქვენ გაიცნობთ ხალხს და გააუმჯობესებთ თქვენს უნარჩვევებს კოდინგის განხრით და სხვა. გარდა ამისა წაიკითხეთ https://learnpythonthehardway.org/ და https://linuxjourney.com/.

დაეხმარეთ სხვებს. თუ თქვენ შეგიძლიათ ასწავლოთ სხვას, ესეიგი თქვენ უკვე აითვისეთ ეს სფერო.

როცა რაიმე ახალს აღმოაჩენთ გააზიარეთ, ამით სხვებიც შეძლებენ მიიღონ სარგებელი. გარდა იმისა რომ თქვენ სხვებს დაეხმარებით, ამასთან ერთად თქვენც უკეთესად შეისწავლით თემას.

გაიღიმეთ თუ შეგიფასებენ ნაშრომს დ შეეცადეთ ეს გამოიყენოთ თქვენს სასარგებლოდ.

bug bounty საზოგადოება შედგება უამრავი ხალხისგან ვინც მზად არის დაეხმაროს სხვებს, ამიტომ ნუ გაგიკვირდებათ თუ რომელიმე მათგანი დაგიკავშირდებათ თქვენი აღმოჩენის ან ნაშრომის განსახილველად. ისწავლეთ თქვენს შეცდომებზე და გამოიყენეთ ეს შეცდომები თქვენს ინტერესებში. მე მაქვს პატარა ბლოკნოტი სადაც ვაწარმოებ პატარა ჩანაწერებს სადაც წერია დღის განმავლობაში თუ ვინ დამიკავშირდა და დამიწერა დადებითი შეფასებები.

ისწავლეთ თუ როგორ მიაღწიოთ მიზანს.

პირველი ნაბიჯია სამიზნის შესწავლა, დაზვერვა და ინფორმაციის წინასწარი შეგროვება. თუ სამიზნე web აპლიკაციაა, დაიწყეთ აპლიკაციის დათვალიერება როგორც ჩვეულებრივი მომხმარებელი და გაეცანით web საიტს და მის დანიშნულებას. ამის შემდეგ დაიწყეთ endpoint-ების, სუბ დომენებისა და პორტების ენუმერაცია.

ერთხელ მეტყევეს კითხეს, "რას გააკეთებდით თუ ხის მოსაჭრელად გაქვთ მხოლოდ 5 წუთი?" მან უპასუხა, "პირველ 2 ნახევარ წუთს დაუთმობდი ცულის გალესვას."

პროგრესთან ერთად თქვენ დაიწყებთ მნიშვნელოვანი ასპექტების შემჩნევას და ჩამოაყალიბებთ საკუთარ მეთოდოლოგიას. შესაძლოა დაიწყებთ სხვა და სხვა მოქმედებების ავტომატიზაციას.
